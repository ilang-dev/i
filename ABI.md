`i-core` exposes the small front-end ABI in `core/i.h`. Front-ends pass
borrowed dense CPU `float32` tensors into `i_tensor`; zero-copy outputs use
`i_output_shapes` followed by caller-allocated `i_tensor_mut` buffers and
`i_exec_into`.

The generated-kernel ABI below is private to `i-core`. It is the ABI for the
dynamically linked libraries generated by 𝚒.

``` c
typedef struct { const float* data; const size_t* shape; size_t rank; } Tensor;
typedef struct { float* data; const size_t* shape; size_t rank; } TensorMut;
size_t count(void);
void ranks(size_t* ranks);
void shapes(const Tensor* inputs, size_t** shapes);
void exec(const Tensor* inputs, TensorMut* outputs);
```
